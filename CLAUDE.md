## What this is

`lumen-ui` is a **token-driven, themeable design system for [egui](https://github.com/emilk/egui)** (Rust).
Widgets read a **recipe** resolved by a **theme** from design **tokens** for a given
`(variant, state, density)`. Swap the theme → the whole app restyles, zero logic changes.

- **Target:** egui 0.34.3, MSRV Rust 1.92. License MIT OR Apache-2.0.

## Golden rule — never hallucinate the egui API

The #1 risk (ROADMAP §E): committing against an egui method that doesn't exist. The original
multi-AI design code did exactly this. **Verify every egui signature against docs.rs AND by
local compilation before commit.** Examples already corrected: `Button` has no
`.padding()`/`.shadow()` (use `Frame`); `get_persisted` needs `data_mut`; `style_mut` →
`global_style_mut`; eframe 0.34 `App` requires `ui()` not `update()`.

## Architecture

Workspace from day zero (ADR-0001). Dependencies flow **down** toward `lumen-ui-core` only.

| Crate | Role | Status |
|-------|------|--------|
| `lumen-ui-core` | tokens, density/context, `Theme` trait + recipes, `install()`, `anim` | v0.1 ✅ |
| `lumen-ui-widgets` | themed widgets (`Button`) | v0.1 ✅ |
| `lumen-ui` | façade: re-exports, prelude, feature flags | v0.1 ✅ |
| `lumen-ui-motion/layout/patterns/icons/themes/material` | created at their roadmap version | v0.4+ |

`lumen-ui-core::theme` is the **frozen core** — recipe/trait signature changes need an ADR.
Full detail: [ARCHITECTURE.md](ARCHITECTURE.md), [docs/glossary.md](docs/glossary.md).
